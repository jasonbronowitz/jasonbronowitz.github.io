AI Architecture Lecture

The AI Architecture

Analogous to Computer Architecture

What if I tell you that every time you type into ChatGPT or Claude, you basically operating a whole computer? This lecture gonna show you how the things that happen inside a traditional computer is really similar to what happens inside a Large Language Model.

01

The Big Picture

Click any layer to see how the computer part morph into its AI equivalent.

I/O DevicesChat Interface
Input / Output
Keyboard & Monitor ↔ Your Text Window
In a computer, the keyboard send electrical signals that get converted to ASCII codes. In an LLM, your text get converted to token IDs — basically the same thing. Output is reverse: token IDs back to text, like GPU turning pixel data into screen image.
CPU / ALUTransformer Engine
Processing
Arithmetic & Logic ↔ Attention & Inference
The CPU use logic gates for arithmetic and decisions. The Transformer use matrix multiplication and attention scores to do the same — weigh inputs and produce output. The ALU is to CPU what attention heads are to the Transformer.
RAMContext Window
Working Memory
Temporary Fast Storage ↔ Current Conversation
RAM hold what the CPU is actively working with. Context window hold everything the LLM is considering right now. Both are fast, temporary, and limited. When they fill up, things slow down or get dropped.
L1/L2/L3 CacheKV Cache
Speed Memory (Cache)
Recently Used Data ↔ Pre-computed Attention Pairs
CPU cache store recently used data close to the processor so it don't gotta go back to slow RAM. The KV Cache store previously computed Key-Value pairs so the Transformer skip recalculating attention for old tokens. Same concept — avoid re-doing expensive work.
Hard Drive / SSDModel Weights
Long-Term Storage
Persistent Files ↔ Learned Knowledge in Billions of Parameters
Hard drive keep files permanently even when power off. Model weights keep knowledge permanently between conversations. The difference? You can easily add a file to a hard drive. Adding knowledge to an LLM require expensive retraining.
Key Insight
A computer and an LLM both solve the same fundamental problem: take input, process it using some knowledge, keep track of what you doing, and give back useful output. They just solve it in very different ways.

03

The Train Window Analogy

Imagine you sitting on a train, looking out the window. The scenery that pass by is like tokens in a conversation. The window got a fixed size — that is your context window. Once scenery scroll past, it gone.

CONTEXT WINDOW
0 tokens sent

Why This Matters
This is why a chatbot "forget" things in long conversations. Its window just got full. Old scenery — your earlier messages — scroll out of view. You gotta remind the AI what you talked about, just like you can't describe scenery you already passed on the train.

Prompt Caching
When thousands of conversations share the same system prompt, prompt caching store the pre-computed KV pairs once and reuse them — like L3 cache shared between CPU cores. First conversation compute it, every conversation after get a cache hit and skip the work. That is why cached tokens cost less on the API.

The Prompt = A Boot Sequence

A good prompt follow the R.C.I.S.F framework — Role, Context, Intent, Specification, Format. Each one map to a layer of the computer boot sequence. Build one interactively below.

① ROLE — BIOS / Firmware
Click to set: "Act as a senior veterinarian with 20 years experience"
→ Loads FIRST into KV Cache. Sets identity. Every future token attend back to this.
② CONTEXT — OS / Environment
Click to set: "Writing a blog for first-time puppy owners with no medical background"
→ Configures audience, situation, and purpose. Like loading the operating system.
③ INTENT — Opcode / Instruction
Click to set: "Write a 500-word article"
→ The main command. WRITE, SUMMARIZE, COMPARE — each activate different processing paths.
④ SPECIFICATION — Operands / Data
Click to set: "Top 5 essential vaccinations for puppies in their first 12 months"
→ The data and constraints. Like operands telling the CPU WHAT to process.
⑤ FORMAT — Output Config / Display
Click to set: "Warm friendly tone, include a table, end with one-sentence takeaway"
→ How to present the result. Like configuring the display driver and output mode.

Why the First Lines Matter Most

The first tokens in your prompt get THREE advantages at once — cache priority, attention anchoring, and survival priority. They is the L1 cache of your conversation.

Influence Decay — Position vs Power

Token at position 1 influence every subsequent token. Token at position 1000 influence far fewer. Earlier = more powerful.

Triple Advantage
First lines get: (1) Cache priority — computed first, stay in KV cache longest. (2) Attention anchoring — positional encoding make early tokens like gravity wells. (3) Survival priority — system prompts get protected when context window overflow. The beginning of your prompt is the most valuable real estate you got.

Putting It All Together

An LLM is not some magic black box. It follow the same architectural patterns we been studying in computer science for decades.

Final Thought
Understanding this analogy make you a power user. Context window = RAM means keep conversations focused. Prompts = commands means write them with structure. Weights = hard drive means the AI won't know last week's news. Role first = BIOS first means identity set the foundation for everything. The architecture thinking change how you use AI forever.
